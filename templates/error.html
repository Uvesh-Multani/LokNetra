Suggested Fixes:

{% if "list index out of range" in error_message %}

Face Recognition Error: This error typically occurs when there are issues with face data processing.

  • Ensure at least one employee is authorized and has face data uploaded
  • Check that employee photos are properly uploaded in the admin panel
  • Verify that the face detection model is working correctly
  • Try restarting the application and camera
  • Check if the camera is properly connected and accessible
{% elif "No camera configurations found" in error_message %}

Camera Configuration Error: No cameras are configured in the system.

  • Go to Admin Panel → Camera Configurations
  • Add at least one camera configuration
  • Set the correct camera source (0 for built-in, IP for network cameras)
  • Configure the camera name and settings
{% elif "No authorized employees found" in error_message %}

Employee Authorization Error: No employees are authorized for face recognition.

  • Go to Admin Panel → Employees
  • Authorize at least one employee by setting "Is Active" to True
  • Upload employee photos for face recognition
  • Ensure employee data is complete
{% elif "camera" in error_message|lower %}

Camera Access Error: There's an issue with camera access.

  • Ensure that the camera is connected properly
  • Check the IP address and port settings in the camera configuration
  • Make sure the camera is powered on and accessible from your network
  • Verify that the required permissions are granted to access the camera
  • Try using a different camera source (0, 1, 2, etc.)
  • Consult the camera's manual for troubleshooting steps
{% else %}

General System Error: An unexpected error occurred.

  • Check that all required dependencies are installed
  • Verify that the database is properly configured
  • Ensure all required files and models are present
  • Try restarting the application
  • Check the console/logs for more detailed error information
{% endif %}
